Examples

Example 1: BoxShadow

To add a drop shadow to a <div>, you would use the box-shadow property, like so: 

.HasBoxShadow #myDiv{
  -webkit-box-shadow: 1px 2px 2px #222;
  -moz-box-shadow: 1px 2px 2px #222;
  box-shadow: 1px 2px 2px #222;
}

This applies the drop shadow to browsers that support BoxShadow. For the rest of the browsers, could draw a border for those browsers instead. 

#myDiv{
  border: 1px outset #444;
}

But you don't want the borders to show up for the HasBoxShadow browsers, so we'll add a rule to remove the border for them:

.HasBoxShadow #myDiv{
  -webkit-box-shadow: 1px 2px 2px #222;
  -moz-box-shadow: 1px 2px 2px #222;
  box-shadow: 1px 2px 2px #222;
  border: 0px none;
}

That's it! No need to conjure up a multitude of convoluted hacks and test over and over again on all the browsers you can get your hands on to make sure all of your hacks work but don't step on each other.

Example 2: RGBAColors

All major browsers except IE now support RGBA colors. You can use RGBA colors to create semi-transparent backgrounds, like so: 

.HasRGBAColors #redTint{
  background-color: rgba(253, 0, 0, 0.2);
}

This should give redTint a red tinted background.

But browsers that do not support rgba colors ignore this rule, and so will not have a different background color. But for this example, let's work around the problem by giving them the color it would have been. To do this I just screenshot the red-tinted portion of the page as rendered in Safari and used the eyedropper tool to get the color. So I set that color for these browsers:

#redTint{
  background-color: #532820;
}

Here is our redTint:

Obviously this isn't as nice. If I change the background color, then the color resulting from the tinting will change also. Not to mention it won't show any content behind the <div>. An alternative is to use a semi-transparent png. But I'll leave that as an exercise for the reader.
